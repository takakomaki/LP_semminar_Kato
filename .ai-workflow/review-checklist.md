# AI コードレビューチェックリスト

## 概要
このチェックリストは、複数のAIによるコードレビュー時に使用します。
各AIが異なる視点からレビューを行い、品質を高めます。

---

## 🔍 GitHub Copilot レビュー
**焦点**: リアルタイムコード品質、構文、パターン

### コード構造
- [ ] 関数の責任が単一か（Single Responsibility）
- [ ] ネストレベルが 3 以下か
- [ ] 循環的複雑度が低いか（10 以下）
- [ ] 重複コードがないか

### 型安全性 (TypeScript)
- [ ] すべての変数に型が指定されているか
- [ ] `any` 型の使用を最小化しているか
- [ ] ジェネリック型が適切に使用されているか
- [ ] Union/Intersection 型が正しく定義されているか

### エラーハンドリング
- [ ] try-catch が適切に配置されているか
- [ ] エラーログが十分か
- [ ] エラーメッセージが明確か
- [ ] 予期しないエラーをキャッチしているか

## 🏗️ Claude レビュー
**焦点**: アーキテクチャ、設計パターン、拡張性

### 設計パターン
- [ ] 適切なデザインパターンが使用されているか
  - [ ] Factory パターン

  - [ ] Strategy パターン
  - [ ] その他: ___________
- [ ] 依存性注入が適切か
- [ ] インターフェースが明確に定義されているか

### スケーラビリティ
- [ ] コンポーネント化が適切か
- [ ] 将来の機能追加に対応可能か
- [ ] 技術的負債がないか
- [ ] レイアリの設計が持続可能か

### 再利用性
- [ ] コンポーネント/関数が再利用可能か
- [ ] ユーティリティが正しく分類されているか
- [ ] DRY 原則が守られているか

## 🚀 ChatGPT レビュー
**焦点**: ベストプラクティス、ドキュメント、メンテナンス性

### ベストプラクティス
- [ ] React のベストプラクティスに従っているか

  - [ ] key prop が適切に設定されているか
  - [ ] memo/useCallback の使用が適切か
- [ ] Next.js の推奨事項に従っているか

  - [ ] Route Handlers の設計
  - [ ] Image 最適化

### ドキュメント
- [ ] JSDoc コメントが十分か
- [ ] 複雑なロジックに説明があるか

### メンテナンス性
- [ ] コードが読みやすいか

- [ ] テストが書きやすい構造か

## ⚡ Deepseek レビュー
**焦点**: パフォーマンス、アルゴリズム、効率性

### パフォーマンス
- [ ] 不必要な re-render がないか
- [ ] API 呼び出しが最小化されているか
- [ ] キャッシング戦略が適切か
- [ ] メモリ使用量が最適化されているか

### アルゴリズム効率
- [ ] ソート/検索のアルゴリズムは最適か
- [ ] 時間計算量は O(n) 以下か（必要に応じて）
- [ ] 空間計算量は最小化されているか
- [ ] ループの最適化は十分か

### バンドルサイズ
- [ ] 不要な依存関係がないか
- [ ] Tree-shaking の対象外になっていないか
- [ ] 大きなライブラリが必要か検討したか
- [ ] インポート パスが最適化されているか
