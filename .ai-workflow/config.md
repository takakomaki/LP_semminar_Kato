# AI コラボレーションワークフロー設定

## 🎯 ワークフロー概要

複数のAI（ChatGPT、Claude、Gemini、Grok、Deepseek）と協働してプロジェクトを進めるための標準化されたワークフロー。

## 📌 各AIの役割分担

### GitHub Copilot (VS Code)
- **役割**: リアルタイムコード補完・実装サポート
- **用途**: コード記述、バグ修正、テスト生成
- **操作**: VS Code内で直接使用（Ctrl+I、Ctrl+K）

### Claude (claude.ai or VS Code拡張)
- **役割**: アーキテクチャ設計・複雑な問題解決
- **用途**: システム設計、リファクタリング戦略、詳細な説明
- **特徴**: 長い会話履歴に強い

### ChatGPT (chatgpt.com)
- **役割**: 実装パターン・ベストプラクティス
- **用途**: 共通の問題解決、コード例、ドキュメント作成
- **特徴**: 一般的な知識が豊富

### Google Gemini (gemini.google.com)
- **役割**: マルチモーダル分析・統合
- **用途**: 画面キャプチャ分析、複数ドキュメント処理
- **特徴**: 画像処理能力が高い

### Grok (x.com/grok)
- **役割**: 最新情報・トレンド情報
- **用途**: 最新技術動向、API仕様変更の確認
- **特徴**: リアルタイム情報アクセス

### Deepseek (deepseek.com)
- **役割**: パフォーマンス最適化・数学的問題
- **用途**: アルゴリズム最適化、複雑な計算、効率化
- **特徴**: 計算能力が高い

## 🔄 ワークフロープロセス

### フェーズ 1: 要件定義 (Claude + ChatGPT)
```
1. プロジェクト要件を Claude に入力
2. ChatGPT でベストプラクティス確認
3. 実装計画書作成
```

### フェーズ 2: 設計・検討 (Claude + Gemini)
```
1. Claude でシステムアーキテクチャ設計
2. Gemini で複数ドキュメント比較
3. 設計ドキュメント完成
```

### フェーズ 3: 実装 (GitHub Copilot + ChatGPT)
```
1. VS Code で Copilot を使用して実装
2. ChatGPT でコードレビューサポート
3. 実装完了
```

### フェーズ 4: 最適化 (Deepseek + Grok)
```
1. Deepseek でパフォーマンス最適化
2. Grok で最新技術を確認
3. 最終調整
```

## 📊 進捗管理

### チェックリスト形式
```markdown
- [ ] 要件定義完了 (Claude, ChatGPT)
- [ ] 設計ドキュメント完成 (Claude, Gemini)
- [ ] コア機能実装 (Copilot)
- [ ] テスト実装 (ChatGPT)
- [ ] パフォーマンス最適化 (Deepseek)
- [ ] 最新技術確認 (Grok)
- [ ] コードレビュー (Claude)
- [ ] デプロイ準備 (ChatGPT)
```

## 💡 効率的な使い方のコツ

1. **並列処理**: 複数AIに同時に異なるタスクを依頼
2. **段階的詳細化**: 最初は大まかに、次に詳細に質問
3. **コンテキスト共有**: 各AIに必要な背景情報を先に提供
4. **バージョン管理**: 各フェーズの成果物をGitで管理
5. **フィードバックループ**: 実装結果をAIにフィードバック

## 🛠️ ツール統合

### VS Code 内での操作
```
Ctrl+I   : GitHub Copilot に質問
Ctrl+K   : コード説明・リファクタリング
Ctrl+/   : コメント生成
```

### 外部AI (ブラウザ)
- 別タブで 5 つの AI サイトを開いて並列作業
- 必要に応じてコードをコピー&ペースト
